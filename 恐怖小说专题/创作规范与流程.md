## 一、立项阶段

### 1.1 新书初始化

```bash
# 一键生成标准目录骨架（需在项目根目录 d:\workspace\writer 下执行）
python init_novel.py <书名>
```

生成：书籍简介+大纲模板+写作规范模板+查重报告模板+伏笔台账+章节状态+角色设定目录+监工日志目录+五卷目录。
详见 `新书模板/SKILL.md`。

### 1.2 市场调研必查项

| 维度 | 方法 | 产出 |
|------|------|------|
| 平台热榜Top50 | WebSearch 番茄/起点恐怖分类 | 当前热门方向 + 代表作品数据 |
| 读者偏好变化 | 搜索书评/论坛/知乎/贴吧 | 上升趋势 + 审美疲劳方向 |
| 竞品差异化空间 | 对比5-10部同类作品核心设定 | 差异化切入点 |
| 平台政策风向 | 搜索"平台不收""签约标准" | 避雷指南 |
| 爆款公式 | 拆解3-5部近期黑马的开篇/节奏/断章 | 可复用的结构模板 |

### 1.2 方向决策矩阵

```
优先级排序（按流量×差异化空间）：
⭐⭐⭐ 规则怪谈×无限流           — 流量最大，但需差异化
⭐⭐⭐ 悬疑灵异×民俗风水×上交国家  — 复兴赛道
⭐⭐  惊悚游戏×收容物×脑洞        — 竞争小，独特性强
⭐⭐  诡秘暗黑修仙                — 新蓝海
```

### 1.3 书名公式

```
番茄风格： 《题材/背景：核心卖点/金手指》
示例： 《收容失控：我能看穿一切规则》

要点：
- 主标题2-5字，钩住题材核心
- 副标题点明金手指or差异化能力
- 避免与已签约作品同名（番茄直接拒稿）
```

### 1.4 立项必查重项

- [ ] 书名多平台搜索（番茄/起点/晋江/微信读书）
- [ ] 金手指名称搜索（避免与已完结作品完全同名）
- [ ] 核心概念组合搜索
- [ ] 主角职业+题材组合搜索

**教训**：金手指名称即使功能不同，只要名称与竞品完全相同就是P0风险。立项阶段必须查、必须改。

## 二、平台赛道与市场格局（2025-2026）

### 2.1 番茄 vs 七猫 vs 起点 — 平台差异

| 维度 | 番茄小说 | 七猫小说 | 起点中文网 |
|------|----------|----------|-----------|
| **用户量** | 日活亿级 | 约6亿注册用户 | 付费用户为主 |
| **用户画像** | 下沉市场、碎片化阅读、男频略多 | 女频见长、轻阅读偏好 | 核心读者、深度阅读 |
| **恐怖题材偏好** | 规则怪谈>悬疑灵异>惊悚游戏 | 轻悬疑+职业/家庭背景 | 民俗灵异>无限流>诡秘修仙 |
| **阅读场景** | 通勤/睡前/厕所，单次<15分钟 | 碎片时间，短章快刷 | 沉浸式，单次>30分钟 |
| **盈利模式** | 广告分成，免费阅读 | 广告+部分付费 | 付费订阅 |
| **推荐机制** | 算法驱动，追读率是核心指标 | 编辑推荐+算法混合 | 编辑推荐+月票榜 |
| **更新要求** | 日更4000+，断更3天掉推荐 | 日更4000+，5天达标进流量池 | 日更4000-6000，稳定压倒一切 |
| **签约门槛** | 1万字可签，首章读完率>40%晋级 | 需内投6000-2万字+大纲 | 3万字+编辑审核 |

### 2.2 恐怖题材赛道热度矩阵

```
🟢 高热度（竞争激烈但流量最大）：
  ⭐⭐⭐ 规则怪谈×无限流×国运直播  — 番茄巅峰榜TOP50占比超40%，新人首选
  ⭐⭐⭐ 中式恐怖×民俗风水×上交国家 — 复兴赛道，《捞尸人》带火
  ⭐⭐  悬疑脑洞×身份反转            — 《诡异邻居来自夸夸群》类爆款

🟡 潜力赛道（竞争小、差异化空间大）：
  ⭐⭐  惊悚游戏×收容物×脑洞          — 已在做的方向，坚持差异化
  ⭐⭐  直播算命×灵异                 — 《直播算命：大哥你媳妇在坟里睁眼了》
  ⭐    诡秘暗黑修仙                   — 新蓝海，但读者教育成本高

🔴 饱和/红线：
  ❌ 纯丧尸/纯鬼怪 — 审美疲劳严重
  ❌ 涉政、黑道、校园暴力、详细犯罪描写 — 直接封书
  ❌ AI生成内容 — 2026年所有平台严查，永久封禁
```

### 2.3 七猫平台特别说明

七猫以女频见长但悬疑推理有忠实读者群。恐怖题材在七猫需走**"轻悬疑"路线**——结合职业/家庭/情感背景，紧张但不血腥。偏血腥暴力的纯恐怖不适合七猫调性。

- 七猫转化率要求≥20%，比番茄更重读者留存质量
- 均留率>5%才获全勤（10万字门槛）
- 前10章完读率≥8%才建议开启首秀（8-15万字时）

## 三、大纲设计规范

### 3.1 总体结构（50万字/5卷）

```
每卷：约10万字，45-50章
每章：2100-2300字
总章数：225-250章
```

### 3.2 卷级结构模板

```
卷名（2字动词/名词，暗示本卷核心冲突）

卷首：前卷状态承接（不直接复述，通过角色行为/对话自然带出）
  ├── 第1-X章：本卷核心异常体/事件引入 + 新角色登场（如有）
  ├── 第X-Y章：事件升级 + 角色关系推进 + 伏笔埋设
  ├── 第Y-Z-2章：高潮——核心冲突解决
  ├── 第Z-1章：收束——代价揭示
  └── 第Z章：超级钩子——必须让读者翻到下一卷
```

### 3.3 跨卷结构铁律

- **每卷有独立核心冲突**（一个主要异常体/反派），同时推进主线
- **卷间由超级钩子衔接**（第一卷结尾的照片地址 → 第二卷追查 → 第三卷何也知道真相）
- **全书核心反转**从第一卷开始埋线，第四卷揭示
- **每卷至少一个超级爆点**（角色身份反转/世界观真相/情感重击）

### 3.4 伏笔系统设计

> 详细规范和操作流程见 `伏笔管理/SKILL.md`（horror-foreshadowing Skill）。
> 台账模板见 `伏笔管理/伏笔台账模板.md`。

**铁律**：
1. **埋设即登记** — 新伏笔必须在台账中分配编号（F001, F002...）
2. **每章对照** — 写前查台账（哪些该推/该收），写后更新台账（推了什么/收了什么/新埋了什么）
3. **超期预警** — 计划回收但超10章未推进的伏笔自动预警

**伏笔台账**（立项后、大纲确定时建立，使用模板 `伏笔管理/伏笔台账模板.md`）：

| 编号 | 类型 | 伏笔内容 | 埋设章 | 计划回收章 | 实际回收章 | 状态 | 关联角色 | 关联伏笔 |
|------|------|----------|--------|------------|------------|------|----------|----------|
| F001 | 跨卷级 | [一句话描述] | 第001章 | 第040章 | — | 🌱 | — | — |
| F002 | 卷级 | [一句话描述] | 第003章 | 第010章 | — | 🌱 | 主角 | — |

**类型**：跨卷级（跨越2卷以上）/ 卷级（本卷内）/ 章节级（1-3章内）
**状态**：🌱未回收 / 🌿推进中 / ✅已回收 / ❌废弃

完整台账包含六个区域：伏笔主表 + 推进日志 + 每章对照检查 + 卷级汇总 + 风险预警 + 废弃记录。

## 四、角色设计体系

### 4.1 角色设计顺序

```
主角（horror-protagonist）
  → 反派体系（horror-antagonist）
    → 配角团队（horror-supporting-characters）
      → 怪物/异常体（horror-monster-entity）
```

### 4.2 主角设计必含项

| 维度 | 内容 |
|------|------|
| 创伤内核 | 具体事件 + 行为体现（不是"他很痛苦"，是"他每次锁车门是因为……"） |
| 性格缺陷三层 | ①表面防御机制 ②情感回避模式 ③深层自我认知扭曲 |
| 金手指 | ①功能边界（能看到什么+不能看到什么）②生理代价 ③五卷进阶/衰退路径 |
| 关系网络 | 至少5条关系线：兄妹/爱情/师徒/镜像/良心，每条有不同情感类型 |
| 识别特征 | 身体标记 + 行为习惯 + 口头禅 + 特殊物品 + 反差设定 |
| 成长弧光 | 每卷起点状态→终点状态→关键转折 |

### 4.3 配角团队配置（5人模板）

| 角色 | 功能 | 与主角关系 | 结局 |
|------|------|-----------|------|
| 搭档/女主 | 医疗+情绪支柱 | 信任重建的第一块砖 | 牺牲 or 存活（二选一） |
| 师父/引路人 | 安全屋+经验传授 | 父亲替代/人间温度 | 建议存活（反套路） |
| 灰色角色 | 侦察+镜像 | 不信任→被迫合作→终极战友 | 存活，角色转型 |
| 新人/弱者 | 良心+关键钥匙 | "你还记得为什么开始吗" | 存活，传灯者 |

### 4.4 反派分层设计

```
第一卷：个人级反派（执念者/扭曲者）— 被主角击败
第二/三卷：理念级反派（极端主义者）— 自我毁灭
第三/四卷：组织级对抗力量（有合理性的"恶"）— 被瓦解/认输
第五卷：存在级终极存在（无恶意，只是"存在"）— 共存的解决方案
```

**核心原则**：每个反派的动机要有让读者犹豫"TA说的也有道理"的瞬间。不用"童年创伤导致变坏"。

### 4.5 怪物/异常体设计规则

```
规则链条公式：
触发条件 → 异常现象 → 规则约束 → 漏洞点

恐怖美学类型（5选1）：
- 日常异变恐怖（同声电话）
- 身体异化恐怖（多余一具）
- 记忆恐怖（忘名楼）
- 身份恐怖（镜面办公室）
- 宇宙恐怖（初始异常）

核心原则：
✅ 异常体 = 规则的bug，不是鬼怪
✅ 每个异常体有隐喻（社会恐惧的具象化）
✅ 恐怖来自规则本身，不来自血腥跳吓
✅ 解释越少越恐怖——留白
```

## 五、多Agent并行写作规范

### 5.1 总原则

**一个Agent = 一卷**。5卷5个Agent，完全并行，互不等待。

### 5.2 写作前必给Agent的材料

每Agent启动前必须提供：
1. ✅ 完整大纲（含所有5卷，让Agent知道自己的位置）
2. ✅ 统一写作规范（人物声口/异常体描写/去AI味清单）
3. ✅ 所有角色设定卡（主角+配角+反派）
4. ✅ 所有异常体设计文档
5. ✅ 本卷专用：章节结构表 + 卷首状态 + 卷尾目标状态 + 与前卷衔接钩子

### 5.3 保证卷间连贯性的方法

```
关键：给每个Agent明确指定——
① 本卷开始时每个角色的状态（承接前卷）
② 本卷结束时每个角色的状态（为下一卷做准备）
③ 必须埋设的伏笔 + 必须回收的伏笔
④ 卷尾超级钩子的具体内容
```

**执行工具**：使用 `horror-chapter-state` Skill 进行卷级状态传递。
详见 `章节状态追踪/SKILL.md`。

### 5.4 章节状态追踪（真相系统）

> 长篇写到50章后崩掉的根源：角色状态漂移、伏笔遗漏、规则自相矛盾。
> "真相系统"是解决这一切的机制——每章写后立即结算，快照是唯一真相来源。

**核心操作**：
```
写前：读取快照 → 读取伏笔台账 → 读取情绪债务 → 输出写前检查报告
写后：更新总账 → 更新快照 → 更新情绪债务 → 输出写后结算报告
```

**七个追踪维度**：
1. **总账**：每章关键事件一行
2. **角色状态快照**：每个活跃角色的最新状态（位置/能力/情感/关系/道具）
3. **伏笔状态快照**：进行中+近期回收的伏笔（联动伏笔台账）
4. **世界观揭示进度**：规则逐步揭示的节奏控制
5. **情绪债务看板**：悬而未决的悬念/冲突/情感承诺
6. **卷级状态传递**：多Agent并行写作的交接面
7. **矛盾日志**：发现的前后矛盾

详见 `章节状态追踪/章节状态模板.md`。

### 5.5 写作上下文打包

> 写第N章时，Agent必须同时看到：大纲定位+角色快照+伏笔提醒+前章承接+写作规范。
> 手动传递这些文件容易遗漏——用工具自动打包。

**用法**：
```bash
python pack_context.py <小说名> <章节号>
```

**打包内容六大模块**：章节位置、角色状态、伏笔提醒、前章承接、写作规范、写作指令。
输出一份结构化Markdown，直接喂给写作Agent。详见 `上下文打包/SKILL.md`。

### 5.6 一致性自动校验

> 长篇写到后面不像同一本书，根源在于缺乏系统化的一致性检查。
> 手工逐一检查200章不可行——先跑自动化脚本做模式检测，再做人工深检。

**用法**：
```bash
# 全项检查
python check_consistency.py <小说名>

# 指定检查项
python check_consistency.py 残响审判 --checks chars,timeline
```

**七项检测**：角色出场追踪、名称一致性、时间线标记、伏笔台账交叉校验、章节结构异常、关键道具追踪、规则关键词矛盾。
输出一份结构化Markdown报告。详见 `一致性校验/SKILL.md`。

### 5.7 监工巡查机制

```
频率：每2小时自动巡查一次
检查项：
  - 各卷目录下章节文件数量（对比目标45-50章）
  - 文件大小（单章不低于3KB ≈ 1500字）
  - 停滞检测：4小时内章节数未增加 → 标记需重启

工具：CronCreate 持久化定时任务
```

### 5.5 Agent续跑策略

当某卷Agent提前结束但章节不足时：
- 启动续写Agent，指定"已有章节范围 + 需补齐范围"
- 续写Agent必须**先读取所有已有章节**再开始写
- 续写完成后检查章节总数

## 六、写作质量标准

### 6.1 每章结构（2100-2300字）

```
0-300字   → 钩子（承接上章悬念 or 新冲突引爆）
300-800字  → 展开（异常体规则感知 / 角色情感推进）
800-1500字 → 第一个小高潮（发现漏洞 / 角色决策 / 冲突升级）
1500-章尾  → 悬崖急刹车（新信息 / 未解悬念 / 反转预告）
```

### 6.2 番茄平台数据标准

| 指标 | 标准 |
|------|------|
| 首章读完率 | ≥40% |
| 完读率 | ≥65% |
| 每500字悬念密度 | ≥1个小悬念或信息增量 |
| 章尾断章率 | 100% |
| 每章情绪峰值 | 1-2次（震惊/反转/碾压/新危机） |

### 6.3 开篇铁律

```
前三行必须有冲突或悬念 → 否则80%读者直接划走
前3章必须展示：
  ① 主角表面性格（建立"这个人有意思"的第一印象）
  ② 主角核心驱动力（钩住情感投入）
  ③ 主角面对异常时的反转（爽点）
第5章必须有第一个大转折
```

### 6.4 恐怖写作核心法则

```
✅ 写"它来过但你看不到它现在在哪"（持续焦虑）
❌ 写"鬼出现了"（跳吓依赖）

✅ 恐怖通过身体反应呈现（胃翻搅、手指掐掌心、心率变化）
❌ 模板化表述（"头皮发麻""浑身发冷""惊出一身冷汗"）

✅ 解释越少越恐怖——只呈现异常，不解释成因
❌ "因为三年前死过人"式科普

✅ 主角会犯错——完美直觉让读者不再担心
❌ 主角智商碾压一切

✅ 每个异常体有隐喻（社会恐惧的具象化）
❌ 纯吓人的怪物（没有深度）
```

## 七、爆款工业化写作技法

> 基于番茄巅峰榜头部规则怪谈/悬疑作品（30+本）拆解提炼。以下技法是数据验证过的工业化流程，不是玄学。

### 7.1 开篇300字黄金法则

番茄平台铁律：**90%读者在前300字决定去留，80%用户3秒内无强钩子直接划走。**

#### 五大开篇钩子类型（选一即可）

| 类型 | 模板 | 实例 |
|------|------|------|
| **信息反差钩** | 常识×异常 碰撞 | "我在殡仪馆守灵，手机弹出死者发来的微信" |
| **危机生死钩** | 直接扔入高压现场 | "解剖刀划开同桌胸腔时，我才发现他心脏还在跳——而我已经死了三天" |
| **规则降临钩** | 诡异规则直接抛出 | "全家入住404号民宿，墙上写着：凌晨三点别照镜子" |
| **身份反转钩** | 你以为的×实际上的 | "系统提示：你已被选为第13位玩家。我看了眼编号——我是第0号" |
| **欲望障碍钩** | 非做不可×巨大障碍 | "妹妹只剩72小时。救她的唯一方法是——通关这个所有人都会死的游戏" |

#### ❌ 开篇五大致命错误（番茄数据验证）

| 错误 | 为什么死 | 修正 |
|------|---------|------|
| 大段环境描写 | 读者来看故事不是来看风景 | 改成刀尖抵喉 + 系统提示音 |
| 多人物群像（首章>2人） | 读者还没记住主角就被分散注意 | 首章出场角色≤2人 |
| 回忆/倒叙开场 | 读者没有理由关心"过去的你" | 直接从危机现场切入 |
| 系统说明式旁白 | 读起来像说明书，不是故事 | 用主角行动展示规则 |
| 主角被动无反应 | 读者代入感来自主角的选择 | 即使吃瘪也必须有明确态度 |

### 7.2 黄金三章模板（规则怪谈/悬疑类）

```
第一章（前500字）：
  ├─ 出主角 + 金手指 + 身份（一句话搞定）
  ├─ 抛出第一条危险规则（不解释为什么）
  └─ 微钩子收尾：主角的第一反应暗示他"不一般"

第二章（500-1000字）：
  ├─ 第一个危机浮现（配角/炮灰触发规则致死——直观展示后果）
  └─ 主角观察到隐藏信息（展示金手指/特殊视角）

第三章（1000-1500字）：
  ├─ 主角利用规则漏洞/金手指破局（第一个爽点）
  └─ 章尾断章：新规则出现 / 倒计时开始 / 更大的异常浮现
```

**自检**：前三章结束后随机抽取3000字，统计"规则揭示/主角优势/危机倒计时"元素数量。**<3个则悬念密度不足，必须修改。**

### 7.3 悬念设计"三步剥洋葱"法

这是番茄规则怪谈类最核心的技法，来自对《规则怪谈：我能完美利用规则》等30+头部作品的拆解：

| 步骤 | 操作 | 心理效果 |
|------|------|----------|
| **第1层：规则揭示** | 开场直接抛出危险规则，**不解释** | 制造认知缺口→好奇心驱动 |
| **第2层：行动验证** | 通过主角行动（遵守/违反）验证规则效果，**不靠文字说明** | 行动感替代说明→沉浸感 |
| **第3层：代价展示** | 配角（炮灰）触犯规则致死，**直观呈现违规后果** | 恐惧可视化→压力传导给读者 |

```
正确示范（三步剥洋葱）：
  "墙上贴着一张纸条：'不要在走廊里跑。'（第1层：规则揭示）
  林渡停下来。身后传来脚步声——是新来的那个男的，他在跑。（第2层：人在违反）
  走廊尽头传来一声闷响。然后什么声音都没了。（第3层：代价展示，但不直接写死状）
  林渡没有回头。"（主角反应强化恐怖）

错误示范（信息倾倒）：
  "这个诡异空间的规则是：不能跑、不能大声说话、不能回头看。
  规则由XX组织制定，目的是XX，违反了会XX而死，因为XX。"
  → 读者划走。你不是在写维基百科。
```

### 7.4 章尾断章技法（强制追读）

#### 三大断章高压法

| 技法 | 写法 | 心理机制 |
|------|------|----------|
| **倒计时断章** | "距离规则生效还剩3秒。" | 时间压力=点击冲动 |
| **悬念螺旋** | 解完一个规则→立刻引入更恐怖的新规则 | 喘息即危险→持续追读 |
| **短句冲击** | 独立成段的短句收尾，不写完完整句子 | 未完成感=必须点下一章 |

```
正确断章示例：
  "林渡终于找到了答案。
  规则不是要你遵守它。
  规则是——
  
  （章尾，读者必须翻页）
  
  规则是要你成为它。"
  
错误断章示例：
  "今天经历了很多事情，林渡感到非常疲惫。
  他决定先休息一下，明天再继续调查。
  夜深了，窗外的风还在吹。"
  → 没有未完成感，读者直接关APP睡觉。
```

#### 断章黄金位置

永远停在：**答案揭示前的临界点** / **情绪的最高潮** / **倒计时的最后一秒**。关键原则：**小闭环 + 大悬念**结合——单章完成一个小情节闭环（读者不觉得被耍），但保留主线更大危机（必须点下一章）。

### 7.5 节奏密度公式（数据验证）

| 密度指标 | 最低要求 | 检测方法 |
|----------|---------|----------|
| 微钩子/信息增量 | 每500字≥1个 | 随机抽3000字，数"发生了什么新变化" |
| 章尾悬崖 | 每章100% | 逐章检查最后一句话是否有悬念推力 |
| 情绪峰值 | 每章1-2次 | 统计震惊/反转/碾压/新危机节点 |
| 大爽点 | 每5章≥1个 | 打脸反派/解锁新技能/身份曝光/关键真相揭示 |
| 小爽点 | 每章≥1个 | 怼人/赚到/获得认可/发现关键信息 |

#### 悬念密度自测法

```
随机抽取连续3000字正文，标记以下三类元素：
  A. 规则揭示（新信息/新限制/新发现）
  B. 主角优势（利用漏洞/展示能力/反直觉操作）
  C. 危机倒计时（时间压力/新威胁浮现/局势恶化）

A+B+C 总数 < 3 → 悬念密度不足，读者会在这3000字区间流失
A+B+C 总数 3-5 → 合格
A+B+C 总数 > 5 → 优秀
```

### 7.6 国运流+直播弹幕框架（新人首选）

```
为什么新人首选国运流+直播：
✅ 主角代表国家利益 → 赢则国赢，输则国输 → 天然绑定读者情绪
✅ 多名玩家参赛 → 配角失败/死亡 → 反衬主角通关的爽感
✅ 直播间弹幕 → 水字数 + 传递情绪 + 放大悬念（"完了完了""主角要死了"）
✅ 成熟框架 → 降低写作难度，先跑通公式再创新

框架配置：
  主角：代表本国参赛，有独特金手指（看隐藏提示/打破规则/读档重来）
  副本：每个国家场景一个副本（中式/日式/欧式恐怖轮换）
  弹幕：每章2-3次弹幕互动，传递观众情绪，制造"直播"临场感
  节奏：单副本3-8章，越来越短，越来越强
```

## 八、市场验证的人物塑造模式

### 8.1 番茄 vs 起点人设差异

| 维度 | 番茄小说 | 起点中文网 |
|------|----------|------------|
| **人设风格** | **标签化、功能化**（爽就完了） | **立体化**（有成长潜力） |
| **核心标签** | "护短""记仇""苟王""欧皇""财迷" | 隐忍、智慧、善良底线、核心能力展示 |
| **出彩方式** | 主角独特优势：看隐藏提示/打破规则 | 解决冲突时展现性格 + 金手指初显威 |
| **记忆点要求** | 读者关APP后还记得主角特征 | 读者理解主角的成长逻辑 |
| **配角设计** | 功能清晰、迅速辨识、不抢主角风头 | 可独立成线、有完整人物弧 |

**核心原则**：番茄读者碎片化阅读，每次打开APP需要立刻辨认角色——标签化不是简陋，是移动端阅读的工业设计。

### 8.2 《十日终焉》"先定骨，后和声"群像法

这是目前网文评论界公认最成熟的人物塑造方法论（《十日终焉》获中国小说学会2024年度好小说第一名，人民日报专题评论）：

```
步骤① 定骨：给每个角色一个"身份基调"

| 角色类型 | 身份基调 | 示例 |
|----------|---------|------|
| 理性者 | 用专业术语解释世界 | 章晨泽（律师）：论点论据头头是道，绝对理性 |
| 街头者 | 用身体语言活在当下 | 乔家劲（混混）："冚家铲"当冒号用 |
| 守护者 | 用责任感撑住一切 | 李尚武（警察）：努力牵头救所有人 |

步骤② 和声：用人物小传揭开"身份基调"下的真实层

  章晨泽（律师）：绝对理性 → 揭开 → 原生家庭中伤出的独立却无力
  乔家劲（混混）：街头脏话 → 揭开 → 口头禅被兄弟义气包裹，实为缅怀与伪装
  李尚武（警察）：牵头救人 → 揭开 → 责任感来自心底最深处的执念与赎罪

步骤③ 配角高光时刻设计

  老吕（小商贩）：前期自私油滑 → 关键时刻绝不掉链子 → 名字"碰瓷"吕布却只取谐音
```

### 8.3 主角人设雷区（番茄读者最反感）

| 雷区 | 表现 | 为什么读者厌恶 |
|------|------|---------------|
| **舔狗** | 无条件讨好女/男配 | 弱→读者不代入 |
| **圣母** | 无底线原谅伤害自己的人 | 憋屈→弃书 |
| **窝囊** | 长时间被欺负不反抗 | 不爽→换书 |
| **没感情天才** | 全程智商碾压无内心波动 | 空洞→无法共情 |
| **金手指过大** | 所有问题一键解决 | 无悬念→无聊 |

**解法**：主角可以有缺陷（强迫症/财迷/社恐），但必须在面对核心冲突时**有态度、有行动、有代价**。

## 九、常见失败模式与应对

> 拆解《捞尸人》（前期封神→后期存疑）、《规则怪谈》（结尾争议）等作品的失败教训。

### 9.1 长篇灵异悬疑的"类型异化"陷阱

**症状**：《捞尸人》前期民俗灵异→中期升级打怪→后期仙侠化。恐怖感被"升级体系"消解。

**病因**：当篇幅拉长至数百万字，纯粹的灵异叙事无法持续。作者用"升级打怪"填充——但升级意味着"可知的体系"，而恐怖需要"未知的压迫"。（光明网文艺评论频道专题分析）

**处方（针对50万字/5卷体量）**：
```
✅ 保持恐怖内核：异常体始终是不可完全理解的——不解释成因，不建体系
✅ 用角色关系推进，而非战力升级推进
✅ 每卷换一个恐怖类型（日常异变→身体异化→记忆恐怖→身份恐怖→宇宙恐怖）
   而非换一个"更高级的怪"
❌ 不要出现"等级""战力值""修炼体系"——这是仙侠的，不是恐怖的
```

### 9.2 金手指过大的"悬念消解"问题

**症状**：《规则怪谈：我能完美利用规则》主角SSS级异能让副本难度被稀释，后期靠反复推演而非真正解谜。

**处方**：
```
✅ 金手指有明确边界（能看到什么 + 不能看到什么）
✅ 金手指使用有代价（生理/心理/时间），且代价越来越重
✅ 随着剧情推进，金手指的局限性越来越明显（而非越来越强）
✅ 主角终究要靠"人"的判断做关键决定，不靠超能力
```

### 9.3 配角"工具人化"

**症状**：中后期配角沦为背景板，只提供信息和功能，失去独立人格。

**处方**：
```
✅ 每个配角有自己的欲望目标（与主角目标不一定一致）
✅ 配角会犯错、会背叛、会恐惧——不全是"主角的队友"
✅ 给配角一个主角不在场的独立行动场景（每卷至少一次）
✅ 配角命运与主角行动的真实后果挂钩（好人也会死，但不是随机的）
```

### 9.4 结局崩坏三宗罪

| 问题 | 表现 | 预防 |
|------|------|------|
| **反转无铺垫** | 《规则怪谈》结尾反转被评"毫无铺垫直接崩坏" | 核心反转从第一卷开始埋线，每卷放一个碎片 |
| **强行圆满** | 所有人生还、反派洗白、世界和平 | 恐怖小说的好结局不是"赢"而是"幸存+代价" |
| **解释过多** | 用大段文字解释世界观真相 | 留白30%——最恐怖的永远是不完全理解的东西 |

## 十、去AI味规范

### 10.1 必删项

| AI模式 | 替代 |
|--------|------|
| 被动词过多（"被看到""被感知"） | 改为主动："他看到了""他察觉到了" |
| AI词汇（总而言之/综上所述/值得注意的是/不可否认） | 直接删除 |
| 夸大的象征意义（"仿佛整个世界都在这一刻崩塌"） | 细化为身体反应："他膝盖一软。手撑住了墙。" |
| 情感告知（"他感到一阵恐惧"） | 身体呈现："他的胃在翻搅。手指掐进掌心。" |
| 三段式排比 | 用不规则的、破碎的句子 |
| 过多的连接词（""然而""与此同时""在此基础上"） | 用句号断开，让节奏更脆 |

### 10.2 人物声口原则

- 心理活动用动作和对话间接呈现，不写"XX心想"
- 每个人物有独特的对话节奏（沈渡嘴贫→沉默切换、何也每句≤两行、老余三句一顿）
- 情绪爆发用沉默+动作，不用"眼眶红了""声音哽咽"

### 10.3 AI写作在恐怖小说中的深层缺陷（CRITICAL）

> 以下诊断来自对照实验：取《诡舍》第17-18章祈雨村招待所场景为人类标杆，由Claude以相同场景前提独立写作，逐段对比后提炼出六个可验证差距。详见 `AI对照实验-祈雨村场景.md`。
> 写每一章后必须逐项自检以下六个维度。

#### 缺陷一：信息交付太快（对照实验证据）

**原文怎么做**（刘承峰报告无头人）：
分七步释放信息——①脸上有汗→②"看到无头人"→③宁秋水质疑→④确认→⑤"它往前走"→⑥声音颤抖→⑦"混入人群消失了"。七步，步步加压。

**AI怎么做**：刘承峰一段话全部说完。"我看到了一个东西。一个没有头的人。它混到人群里了。"

**差距**：AI急于把信息交给读者。原文知道"延迟就是悬念"——"然后……"的省略号比内容更重要。

**铁律**：关键恐惧信息的交付至少分三步。每步之间插入角色反应、环境变化或他人打断。不要让角色一次性说完。

#### 缺陷二：过渡描写过度——场景切换不该"解释怎么来的"（对照实验证据）

**原文怎么做**："再一次恢复意识的时候，宁秋水发现自己已经身处一座巨大的建筑面前。" —— 一句完成场景切换。

**AI怎么做**："大巴在浓雾里开了很久。久到宁秋水觉得自己可能已经睡着了又醒来，醒来又睡着。等他终于恢复意识的时候……" —— 三句铺垫。

**差距**：AI不信任读者能跟上场景跳转。原文信任。

**铁律**：血门/副本场景切换，用"恢复意识时，已经身处XX"。不要写"怎么来的"。

#### 缺陷三：用"确定性"杀死恐怖——把"好像"写成"是"（对照实验证据）

**原文怎么写烧焦味**："空气之中，始终透露出一股子……烧焦的味道。好像，有什么东西烧起来了。" —— 两句。"好像"=不确定=恐怖。

**AI怎么写烧焦味**："不是霉味。也不是灰尘味。是一种——烧焦的味道。像是有什么东西在不远的地方被烧过，那种焦糊感渗透进了墙壁和地板。" —— 四句。排除法+确认+细节化。

**差距**：AI每次都想把感觉写清楚。"清楚"恰好是恐怖的反面。"好像有什么东西烧起来了"比"焦糊感渗透进了墙壁和地板"更恐怖——因为前者制造了不确定性。AI还加了一个"不是霉味也不是灰尘味"的推理过程——人在恐惧中不做逻辑排除。

**铁律**：恐怖感觉写得越确定越不可怕。保持"好像""似乎""像是"——不要替换成"是"。

#### 缺陷四：关键时刻用陈述句替代疑问句——把"难道"写成"所以"（对照实验证据）

**原文怎么写血玉预警结论**："难道……整个招待所都不干净？" —— 问号结尾。读者和主角一起停留在不确定中。

**AI怎么写**："整个招待所都不干净。" —— 句号结尾。确认完毕。

**差距**：AI替读者得出了结论。原文让读者和主角一起在疑问中停留。问号=悬念推力。句号=悬念终止。

**铁律**：角色得出恐惧结论时用疑问句，不用陈述句。"难道……"比"显然"更恐怖。

#### 缺陷五：漏掉角色的本能反应——没写"卧槽"（对照实验证据）

**原文怎么写角色发现血玉更亮后的反应**："卧槽！"宁秋水忍不住暗自爆了句粗口。

**AI怎么写**：无。直接进入下一步探索。

**差距**：原文的"卧槽"是全段最有人味的一句话。AI写了信息、写了描写、写了推理——漏掉了人。

**铁律**：每个恐惧发现之后，给角色一个本能的、不拘束的反应——可以是一个词、一个动作、一个停顿。不要跳过。

#### 缺陷六：出不了黑色幽默——恐惧中最"人"的时刻（对照实验证据）

**原文怎么写鬼离开后**："走的时候，它甚至帮宁秋水带上了房门。'这鬼……还挺有礼貌。'宁秋水自嘲式的在心里调侃了一句。"

**AI怎么写**：门被带上→咔嗒→安静→主角看天花板→后背冷汗。没有幽默。

**差距**：黑色幽默在AI写作中是完全缺席的。因为"恐惧+幽默"的组合在逻辑上矛盾——而AI优先选"合理"的反应。但人类在极度恐惧后会本能地用黑色幽默重建控制感。这不是技法的缺失，是"理解角色到可以替他开玩笑"的能力缺位。

**铁律**：最恐惧的场景之后，允许角色在内心开一个不合时宜的玩笑。这不会破坏恐怖气氛——它让角色从"恐怖小说的受害者"变成"读者关心的人"。

### 10.4 AI味自检清单（每章写完后逐项打勾）

> 以下8项来自对照实验的六个可验证差距。每写完一章，逐项检查。

```
[ ] 1. 信息交付延迟：关键恐惧信息是否至少分三步释放？有没有一次性说完整段话的对话？（对照实验缺陷一）
[ ] 2. 场景跳转：是否用"恢复意识时，已经身处XX"直接切入？有没有写"大巴开了很久/走了一段路/昏过去了又醒来"？（对照实验缺陷二）
[ ] 3. 恐怖感觉的不确定性：搜索本章所有"是""一定是""显然""确定"。能不能替换成"好像""似乎""像是""难道"？（对照实验缺陷三/四）
[ ] 4. 恐惧结论的句式：角色得出恐惧结论时，用的是问号（"难道……？"）还是句号（"整个地方都不干净。"）？（对照实验缺陷四）
[ ] 5. 本能反应：每个恐惧发现之后，角色有没有发出一个本能反应（一个词/一个动作/一个停顿）？还是直接进入下一步分析？（对照实验缺陷五）
[ ] 6. 不合时宜的幽默：最恐惧的场景之后，角色有没有在内心开一个黑色玩笑？如果没有——这个角色还像人吗？（对照实验缺陷六）
[ ] 7. 意象原创性：搜索高频AI恐怖词（头皮发麻/毛骨悚然/不寒而栗/汗毛倒竖/后背发凉/惊出一身冷汗）。每找到一个，替换为：身体反应+动作 或 原创意象 或 不写。
[ ] 8. 情感克制：情感最重的段落超过50字了吗？超过了，砍到50字以内。
```

## 十四、核心经验总结

### 来自《收容失控》完本经验

1. **金手指名称是P0查重项**——与已完结作品撞名是直接拒稿风险，立项必须查、必须改
2. **卷间衔接钩子是连惯性关键**——每卷结尾必须有一个让读者必须翻下一卷的悬念
3. **并行写作的前提是充分的前期准备**——大纲+人设+写作规范越细，Agent跑偏越少
4. **Agent不是一次性能写完50章的**——预期2-3轮续写补齐，监工巡查机制是保障
5. **异常体=现实规则bug**这个定义是全书最大差异化资产——每个异常体都能在这一定义下衍生出独特的恐怖美学
6. **反套路结局（共存而非消灭）是读者记忆点**——在"收容/消灭"的主流范式中做出区分
7. **配角不死光**——老余和何也的存活让结局不是"惨胜"而是"疲倦但值得"
8. **章节统一零填充数字命名**——中文数字在大批量文件中排序混乱，教训

### 来自市场调研的新教训（v2.0）

9. **番茄前300字决定80%读者去留**——开篇必须信息反差钩/危机生死钩/规则降临钩三选一，大段环境描写=自杀
10. **恐怖≠跳吓，恐怖=持续焦虑**——写"它来过但你看不到它现在在哪"，不写"鬼出现了"。从《捞尸人》教训看：灵异悬疑长篇化的最大敌人是"升级体系消解未知感"
11. **章尾断章是追读率的工业机器**——每章结尾停在答案揭晓前的临界点。数据验证：断章率100% = 追读率≥65%
12. **金手指有边界比金手指大更重要**——《规则怪谈》的教训：SSS级异能让主角靠推演而非解谜，稀释了规则恐怖的魅力。金手指必须随剧情暴露越来越多局限性
13. **番茄读者要标签化人设，不是立体化人设**——碎片化阅读场景下，读者每次打开APP需要立刻辨认角色。标签不是简陋，是移动端工业设计
14. **长篇恐怖的核心矛盾**——恐怖需要"未知的压迫"，但长篇需要"可知的体系"让读者持续投入。解法：不建战力体系，用角色关系推进；每卷换恐怖类型而非升级怪物
15. **新人先跑通公式再创新**——国运流+直播+规则利用是当前最成熟的番茄恐怖框架。先做到月入过万，再融入个人风格。日更稳定性 > 爆发更新
16. **结局留白30%**——最恐怖的水远是不完全理解的东西。反转必须有全篇铺垫，不能结尾突然抛出新设定（《规则怪谈》结局教训）

## 十五、一手文本分析：四大爆款作品技法拆解

> 以下分析基于2026年6月对四部作品原文的逐章阅读。每个结论都标注了具体文本出处，可自行对照原文验证。

### 15.1 开篇第一句话的四种解法

**《规则怪谈：我能完美利用规则》— 石头巨怪**

开篇第一句：
> "杂乱的房间。发黄的墙壁。满地的鲜血、头发、残肢……"

技法：**碎片句堆叠 → 感官轰炸**。三个名词短语，不用动词。读者被直接扔进犯罪现场般的空间，没有"从前有座山"式的引入。第6行才出现主角名字"顾毅"，第12行主角才问"我是在哪儿？"——读者和主角同时被剥夺方位感。

关键决策：**延迟主角自我介绍**。前5行只有环境，迫使读者在认知空白中产生焦虑——这正是恐怖小说的生理基础。

**《十日终焉》— 杀虫队队员**

开篇第一句：
> "一个老旧的钨丝灯被黑色的电线悬在屋子中央，闪烁着昏暗的光芒。"

技法：**单一意象慢描 → 氛围渗透**。一个灯，一根线，一道光。然后"静谧的气氛犹如墨汁滴入清水，正在房间内晕染蔓延"——用比喻将静止空间动态化。到第24行才出现山羊头，第29行才出现主角齐夏的名字。

关键决策：**先建空间，后放人物**。读者先知道"这是一个密闭的、没有门的房间"，然后才知道"有十个人被困在这里"。空间的压迫感先于人际的紧张感。

**《捞尸人》— 纯洁滴小龙**

开篇第一句：
> "「细那康子们，吃饭了，呜嘞呜嘞呜嘞～」"

技法：**方言对话 → 烟火气反差**。一个农村奶奶敲着粥缸喊孩子吃饭。没有任何恐怖元素。直到第185行才出现第一具浮尸——在此之前是纯粹的家庭生活场景：孩子抢粥、奶奶骂人、白事班子八卦、撑船摘莲蓬。

关键决策：**用185行纯生活铺垫一处恐怖**。这在番茄平台是自杀行为，但在起点是顶级技巧——恐怖的有效性 = 正常生活的厚度 × 异常侵入的烈度。读者对李追远的关心建立在185行的日常细节之上。

**《诡舍》— 夜来风雨声**

开篇第一句：
> "浓雾中，一辆破旧的大巴正沿着一望无际的公路缓缓驶来。"

技法：**经典哥特式开场 → 渐进恐怖**。雾、无人驾驶、未知目的地——三个元素在首句全部到位。第25行"驾驶位上……空空如也"确认无人驾驶。这种"如常场景中的一个反常元素"是恐怖小说最古老也最有效的开局。

关键决策：**用闪回延迟第一恐怖**。胖子被剥皮的故事不是实时发生的，而是通过回忆讲述的。这降低了即时冲击力，但增加了"现在我们也可能死"的持续性焦虑。

### 15.2 人物出场的文本技法对比

四部作品的主角出场方式截然不同，但有一条共同规律：**没有一个作者用旁白描述主角**。

| 作品 | 主角首次出现 | 技法 | 读者获得的信息 |
|------|------------|------|--------------|
| 《规则怪谈》 | "刺鼻的血腥味将顾毅唤醒，他挣扎着从地上爬了起来，看着手心怔怔地发呆。" | **处境反应** | 他在危险中、他困惑、他没有特殊能力 |
| 《十日终焉》 | "一位名叫齐夏的年轻人坐在距离山羊头最远的地方，他迅速打量了一下屋内的情况，片刻之后，神色就凝重了起来。" | **观察者姿态** | 他坐得最远（警惕）、他主动打量环境（聪明）、他能发现异常（能力强） |
| 《捞尸人》 | "最后一个过来的是一个十岁的小男孩，他身穿牛仔吊带裤，脚上是时兴凉鞋，皮肤白嫩，面带腼腆。" | **环境反差** | 他与周围孩子不同（城市来的）、他腼腆（性格内向） |
| 《诡舍》 | "关键时候，人群中的一名瘦削男人开了口，他戴着一个深褐色的方框木制眼镜，清秀的面容上彰显着与他人不一样的冷静。" | **行动定义** | 他是第一个开口的人（领袖）、他戴眼镜（知识分子）、他冷静（心理素质好） |

**共同规律**：人物 = 特征行为 + 环境关系 + 他人反应。三者的组合顺序可以不同，但**三者必须都在首次出场的前200字内出现**。

### 15.3 规则/信息交付的文本机制

恐怖小说的核心是"让读者知道有规则，但不完全知道规则是什么"。四部作品用了三种不同的机制来交付信息：

**机制一：《规则怪谈》的"模拟推演"——最具创新性**

```
原文结构：
（推演开始！）
（你第一次使用天赋技能，心情激动。）
（你从床上爬了起来……）
（你死了。）
（推演结束。）
```

这个机制的价值远不止"让主角多次死亡"。逐段分析：

- **信息节奏**：每次推演约100-200字，揭示1-2条新规则。读者和主角同步学习。
- **恐怖复用**：主角死了5次（前两章），每次死法不同——被医生杀、被护士戳眼、天花板滴血致死、被厨师劈头。死亡不再是"终局"，而是"学习工具"。
- **括号体的文本功能**：括号内使用第二人称"你"，制造"读者=主角"的代入错觉。括号外的第三人称叙事保持客观距离。两层叙事的交替创造了独特的阅读节奏。

**机制二：《十日终焉》的"一句话信息炸弹"**

> "早安，九位。"山羊头率先说话了。

十个坐着的人，山羊头说"九位"。这个信息在整个第一章中只出现这一次，但它是驱动全部悬念的核心。齐夏反复思考"九位是什么意思"，读者的脑子也跟着转。

技法要点：**在一个看似正常的句子中埋一个不正常的数字/事实，不解释、不强调、不重复，让读者自己去发现**。

**机制三：《捞尸人》的"日常对话嵌恐怖"**

> "这河里可是有水猴子的，专拉人下水淹死做自己的替死鬼"
> ——李维汉对孩子说的玩笑话
> 185行后：河里漂着一具真正的浮尸

技法要点：**恐怖信息先以"假"的形式出现（大人吓小孩的故事），然后被证实为真。读者经历了从"不信"到"信"的认知反转，恐怖感加倍。**

**机制四：《诡舍》的"闪回展示代价"**

> "那红绿灯上一直飘扬着的……赫然是先前离开的胖子血淋淋的人皮！那张皮被完整地剥离，众人甚至还能看见胖子的表情极度惊恐扭曲……"

技法要点：**不写死亡过程，只写死亡结果。读者的想象力会自动填充最恐怖的过程。**

### 15.4 章尾断章的真实文本分析

逐章检查了四部作品前5章的结尾方式：

| 作品 | 第一章结尾 | 断章类型 |
|------|----------|---------|
| 《规则怪谈》 | "（你翻到纸条背面，上面有人用血写下了一行文字。）（"不要吃他们给出的任何药品，他们是一群魔鬼，他们会将你引入深渊。"）" | **新信息反转** —— 正面规则 vs 背面血字 = 矛盾 |
| 《十日终焉》 | "能够将人类最坚硬的头骨用手在桌面上撞碎，那个山羊头还是「人」吗？" | **不可解之问** —— 不是剧情悬念，是存在层面的恐惧 |
| 《捞尸人》 | （无断章——章尾自然收束到下一段日常对话） | **无钩** —— 起点允许这种写法，番茄不允许 |
| 《诡舍》 | "咯咯——木门内，像是有什么东西在推门。很快，木门被一双苍白的手推开了！" | **物理临近** —— 门开了，恐怖物即将出现 |

**核心发现**：番茄平台的断章技巧不是"留悬念"那么简单。有效的番茄断章 = **可操作的信息 + 不可预测的后果**。读者知道自己知道了什么，但不知道知道了之后会发生什么。

《规则怪谈》章尾："纸条正面说吃药，背面血字说不吃药"——这是可操作信息（两条矛盾规则）+ 不可预测后果（选错了就会死，但主角有推演可以试错）。

《诡舍》章尾：门开了——这是可操作信息（马上就要进去）+ 不可预测后果（里面有什么？任务是什么？）。

**断章失败案例**（来自《诡舍》后期章节101章结尾）——章尾是"李真哑声说道：对不起，老师。"然后自然结束。这是一个**情感收束**，不是**悬念推力**。读者获得了情感满足，没有点下一章的动力。

### 15.5 中后期技法变化追踪

**《规则怪谈》第100章 vs 第1章对比**：

第1章：单一空间（病房）、单一主角（顾毅）、核心机制驱动（推演→死→再推演）
第100章：多线并行（现实世界的曲康平 + 仍在副本中的顾毅）、配角视角主导、能力对抗（灵视之眼 vs 猫脸女人）

**变化**：
- 从"规则解谜"滑向"能力对决"。第100章的核心不是"曲康平如何发现规则漏洞"，而是"曲康平用灵视之眼制造幻象钓鱼执法"——本质是超能力战斗。
- 单章2000字的"推演循环"被拉长为多线叙事，单线密度下降。

**《捞尸人》第1章 vs 第100章对比**：

第1章：红薯粥的味道、奶奶的咸酱、撑船的竹篙——感官写实
第100章：堕情泥胎、官将首扶战、白鹤童子上身、龙王秦柳两家——体系化玄幻

**变化**：这是最剧烈的。从"你不知道河里有什么"到"你知道这叫什么、怎么分类、怎么打"——**恐怖被体系消解**。当李追远可以冷静地判断"余婆婆是堕情泥胎，属于死倒自我修复的一种邪术"时，读者不再害怕——他们在学习一个游戏规则。

**《诡舍》第101章 vs 第1章对比**：

第1章：胖子的人皮——通过结果暗示恐怖
第101章：笑男被班主任拥抱后释然——恐怖消解为温情

**变化**：后期的"解谜→和解"模式替代了前期的"解谜→更深恐怖"模式。读者反馈的"恐怖感下滑"从这里可以看出来——不是因为文笔变差，而是因为**恐怖物的"人性化"消解了恐怖**。当笑男只是一个"想妈妈的孩子"、班主任只是一个"失去儿子的父亲"，恐怖变成了悲剧。

### 15.6 四部作品的人物塑造文本技法

**"身份基调"的文本呈现——来自《十日终焉》**

直接看原文，每个人物第一次说话：

```
花臂男："你……是谁？"  → 犹豫、后知后觉。他的"凶"是后来的。
清冷女人："我劝你早点停止自己的行为，我怀疑你拘禁我们已经超过了二十四个小时，构成了「非法拘禁罪」……" → 律师的思维方式和语言习惯在第一句台词中完成建立。
白大褂："我们都刚刚才醒过来，你怎么知道我们被囚禁了「二十四个小时」？" → 医生的诊断思维：捕捉矛盾、质疑断言。
```

技法精髓：**不用"章晨泽是个律师"这样的旁白，而是让她用律师的语言思考。人物的职业/性格不是标签，是语言结构。**

**《捞尸人》的代际人物网**

李追远的核心关系在一个早餐场景中全部建立：
- 崔桂英（奶奶）：偏心但不过分——多给外孙一勺肉酱
- 李维汉（爷爷）：沉默的慈爱——让外孙点烟是最高级别的亲密
- 虎子、石头（表兄弟）：吃醋但无恶意——儿童的自然反应
- 小黄莺（失踪的表演者）：通过大人八卦引入——不在场的人物却推动全部悬念

技法精髓：**用一个吃饭场景同时建立6个人物 + 1个悬念。每个对话都是一石二鸟。**

### 15.7 关键结论：你可以直接用的

**1. 开篇策略按平台选择**

- 如果发番茄：用《诡舍》式开局（场景 + 异常 + 代价展示，300字内）
- 如果发起點：可以参考《捞尸人》式开局（生活厚度 → 恐怖渐变），但必须有更强的钩子
- 无论哪个平台：**不要像《十日终焉》那样开篇纯氛围描写**——它成功了，但它的成功建立在470万追更基础之上，新人没有这个读者耐心额度

**2. 人物出场的铁律**

旁白介绍人物 = 业余。正确做法：每个人物首次出现时，用一个**不可替代的行为**呈现其核心特征。律师说"非法拘禁罪"，医生质疑时间逻辑，观察者沉默计数。

**3. 规则/信息的交付不要一次给完**

《规则怪谈》的病人手册给出了完美模板：正面6条规则 + 背面1条血字。正面是"官方版本"，背面是"隐藏版本"。信息的矛盾创造悬念。如果你写规则怪谈，找一个能制造"官方 vs 隐藏"二元对立的信息载体。

**4. 章尾断章用"可操作信息 + 不可预测后果"**

不要停在"主角很害怕"或"气氛很恐怖"。停在"主角知道了X，但不知道X会带来什么"。让读者的大脑自动计算可能性——这个计算过程就是点击下一章的动力。

**5. 避免"类型异化"——保持恐怖内核**

从《捞尸人》第100章和《规则怪谈》第100章的变化可以看出：给恐怖物命名、分类、制定打法 = 杀死恐怖。确保你的异常体始终有30%是不可解释的，即使主角掌握了应对方法。
