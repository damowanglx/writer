# 第二卷：《节点战争》

## 第056章 反制

陈渡没有挂断林昭的电话。他站在路口，右手按在把手上，右臂的纹路还在缓慢推进——每十秒一毫米，像一根无声的秒针在皮肤下走。

“苏墨知道我的位置。”他说。不是疑问句。

“对。”林昭的声音很紧。“你写‘位置公开’的时候我没拦住你。我当时想——至少你在#31面前是透明的，这可能是公平的。但我忘了一个人能看到所有规则信号的编译者。”

陈渡想起医院那天。苏墨隔着两条街改变了墙壁的硬度。他不是用眼睛锁定的——他是用规则信号。

“他多久能到？”

“不知道。”林昭说。“他不在城东，但他有自己的移动方式。”

陈渡看了一眼便利店的方向。他还剩几分钟——不是#31给他的时间，是苏墨给他的时间。

他必须在苏墨赶到之前解决#31。

挂断电话，他没有往分拣站骑。他停在原地，闭上眼。

老周说规则可以看见的。

刚才在仓库里他太慌了——纹路在推进，疼痛感在干扰思维，他没来得及真正“看见”什么。现在他站在路灯底下，右臂的黑色线条停在肘上三厘米，呼吸平稳下来。

他重新感知。

空气里的静电还在。他闭上眼睛，把注意力从疼痛上移开——移到周围的空间里。一开始只有路灯的嗡嗡声，远处货车的引擎声，树叶的沙沙声。然后——他感觉到了。

#31的三条规则像三根绷紧的琴弦，悬在分拣站上空。

第一条：数据采集。透明，但厚重。像一张巨大的滤网覆盖了整个区域——所有的快递信息在穿过这层网时被复制。

第二条：终端机禁令。覆盖面小得多——只限定在每台终端机周围大约两米的范围。像一个个透明的罩子扣在机器上。

第三条：加速惩罚。没有固定位置——它是一种“执行逻辑”，像一段等待触发的代码，挂在第二条规则下面。

陈渡睁开眼。

他看见了。不是用眼睛——是用感知在脑子里画出了这三条规则的空间结构。像一个全息投影，清晰到不可思议。

第一条规则覆盖了整个分拣站和周边几个街区。这是#31的“地基”——所有其他规则都建在这上面。

第三条挂在第二条下面。

第二条挂在第一条下面。

三层嵌套。不是平等的三条规则——是一条主规则和两条子规则。如果第一条被覆盖，第二条和第三条就失去了执行依据。

他需要覆盖第一条。

但这比覆盖第三条更难。因为这相当于在#31的领地上，改写他的核心规则。

陈渡看了一下自己的手机。APP还开着。屏幕上的输入框在等他写规则。

他试过写一条采集范围缩小的规则——被拒绝了。

拒绝的原因是“权限不足”。或者更准确地说——这条规则的“深度”不够。

他想起老周笔记本上的一句话：“覆盖的深度取决于你对自己规则的信任程度。你的规则越抽象，它能覆盖的就越多。”

抽象。

不是越精确越好——是越广泛越好。

他的第三条规则是“加速区域内所有编译者的纹路”。这条就比#31的第三条“违反者纹路三倍加速”更抽象——因为他没有指定条件，没有指定倍数，没有指定触发方式。就是“加速”。这句话能涵盖几乎所有的加速场景。

如果能用同样的逻辑写第一条覆盖规则呢？

陈渡站在路灯下，右手食指悬在手机屏幕上方。

他需要写一条能覆盖#31数据采集规则的规则。如果写“此区域禁止数据采集”——太具体，太容易被反覆盖。如果他写——

“此区域所有信息传递——不受追踪。”

他打完了这行字。

手机屏幕闪了一下。规则正在提交处理。

陈渡屏住呼吸。

屏幕没有弹出拒绝。

它在处理——比正常慢了大约三倍。像一台老电脑在运行大程序，CPU在满负荷运转。他能感觉到系统在比较两条规则的“覆盖力”——他的规则比#31的规则更抽象吗？

“此区域所有信息传递——不受追踪。”

#31的数据采集规则只针对快递包裹。他的规则针对“所有信息传递”。范围更广——这在覆盖逻辑上应该更强。

但系统还在处理。

三秒。

五秒。

手机屏幕突然亮了一下——不是规则通过的消息。是#31的直接通信：

“你写了一条覆盖规则。”

陈渡盯着那行字。没有回复。

“你知道你在干什么吗？”

陈渡还是没回复。他在等系统结果。

“十七年。” #31的第二条消息。“我写了十七年的规则。你一晚上的年轻人想覆盖我的主规则？”

屏幕终于弹出了结果：

“规则已提交。等待作者确认。”

等待作者确认——不是“规则已接受”。

陈渡明白了。系统认可了他的规则的覆盖力——但因为是跨作者覆盖，最终决定权在原作者的终端机上。#31可以选择“接受覆盖”或“拒绝覆盖”。

如果是接受——旧规则被降级，新规则生效。

如果是拒绝——两条规则并存，在系统层面产生冲突。

而冲突——会产生裂隙。

#31不会接受。他写了十七年，不会让一个四个月的新人覆盖他。

但裂隙呢？

如果他拒绝——两条规则同时在#31的节点上运行。数据采集规则和“不受追踪”规则——一个是“收集”，一个是“不收集”。它们在语义上完全矛盾。

陈渡站在路灯下。路灯的光照在他的手机屏幕上。

他想到了老周的第三条法则：永远不要在别人的主节点范围内发动攻击。

但他也不是在攻击——他在覆盖。覆盖不是攻击，至少系统的定义里不是。系统把覆盖和攻击分成了两套不同的机制。

他的手机震动了一下。

#31的回复：“我拒绝你。”

紧接着——整个区域的路灯同时熄灭了。

不是断电。是规则层面的熄灭。

陈渡感觉到空气里的压力骤然变化——像沉入深水区。他的耳朵嗡了一声。手腕上的纹路开始重新推进——从缓速变成了中速。

#31的终端机在回应他的覆盖请求——不是接受，是反击。

陈渡站在黑暗里。手机屏幕是唯一的光源。

这不是拒绝。这是战争。

他低着头。看着APP界面里那行等待确认的规则文本。系统还在等#31的正式拒绝操作——但#31已经开始用其他方式回击了。

这意味着什么？意味着#31慌了。

如果他真的自信能拒绝，他不会在系统结果出来之前提前动手。他在害怕陈渡的规则真的覆盖成功。

陈渡突然有了一个念头——不是覆盖，不是删除——是加速。

他打开APP，重新写了一行规则：

“此区域所有编译者的规则——执行速度提升至十倍。”

屏幕亮了。没有等待，没有延迟——直接显示：“规则已生效。”

陈渡愣住了。

为什么这条规则通过了？

因为——这不是覆盖，也不是冲突。这是一条独立的辅助规则，不影响任何已有规则的语义。#31没有理由拒绝它——因为它改变的是执行速度，不是规则内容。

但十倍执行速度意味着什么？

意味着——数据采集规则会以十倍速度采集。终端机禁令会以十倍速度检测。加速惩罚会以十倍速度执行。

也意味着——他刚提交的“不受追踪”覆盖规则——会以十倍速度等待#31的确认。

而十倍速度——系统会认为这是一个需要优先处理的异常状态。

连锁反应开始了。

陈渡看着手机屏幕。

#31的消息：“你疯了。”

陈渡没回。他看着右手上的纹路——因为系统执行速度提升，他的纹路推进速度也在加速。但这不重要。重要的是——#31的拒绝操作如果不在短时间内完成，系统会自动把“等待确认”状态判定为“超时接受”。

他不是在攻击#31。

他是在把#31拖进一个他自己也无法控制的速度里。

陈渡靠在路灯杆上。黑暗里，他笑了。

这是他第一次用自己的规则来修改战斗节奏——不是写一条攻击规则，是改写系统本身的运行参数。

#

便利店仓库的终端机屏幕突然亮了。

不是显示#31的界面——是显示陈渡的规则列表。

他的第四条规则“位置公开”正在以十倍速度运行。这意味着——他在这座城市里的信号亮度，从“亮着的灯”变成了“灯塔”。

林昭的电话又来了。

“你在干什么？你的信号强度突然暴涨——我现在闭着眼睛都能感觉到你在哪！”

“我在加速。”陈渡说。

“加速什么？”

“所有东西。”

电话那头沉默了几秒。然后林昭说了一句让他脊背发凉的话：

“那你得知道——苏墨也在加速朝你过来。”

（第056章 完）
