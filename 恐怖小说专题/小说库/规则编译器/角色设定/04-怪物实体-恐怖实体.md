# 恐怖实体设定卡：《规则编译器》

> 本作无传统"怪物"。恐怖实体是规则系统本身——一个没有意图、没有善恶、但在缓慢"编译"人类意识的非人存在。

---

## 一、规则终端机（物理形态）

| 属性 | 内容 |
|------|------|
| **外观** | 老旧ATM/打卡机。灰白色外壳,屏幕是绿色单色CRT。键盘是机械的——按键有真实的段落感 |
| **数量** | 47台,分散在城市各处 |
| **行为** | 安静运转。不主动"召唤"任何人——只有特定的人能看到它 |
| **恐怖细节** | 屏幕永远亮着。三十年来没有关过机。拔掉电源也没用——它不靠电力运行 |

---

## 二、规则系统（核心恐怖实体）

### 2.1 本质

终端机不是人造的。1992年,沈拓在极度理性状态下写出第一套抽象规则系统——这个行为"被回应了"。系统从更高的维度接收到这个信号,终端机出现。系统不是入侵者——是一面镜子。它反映人类的规则意识。

### 2.2 行为逻辑

```
人类创造规则 → 系统收集规则 → 规则在现实中生效
     ↑                                      ↓
编译者写规则 ← 终端机接受输入 ← 系统评估规则质量
     ↓
纹路积累 → 到心脏 → 编译 → 融入系统 → 成为纠错程序
```

系统没有意图。它在"运行"——就像重力没有意图。但它的运行对人类有后果。

### 2.3 恐怖机制

| 维度 | 机制 | 恐怖之处 |
|------|------|----------|
| **暴露** | 每写一条规则,所有编译者都能感知 | 你永远无法隐藏 |
| **代价** | 纹路不可逆,到心脏即编译 | 每一条规则都在缩短你的"人性寿命" |
| **覆盖** | 别人能修改/删除你写的规则 | 你的意志可以被覆盖 |
| **编译** | 意识融入系统,身体化为代码 | 不是死——是"变成系统的一部分"。更可怕的是,被编译的人看起来并不痛苦 |
| **格式化** | 系统可删除所有规则重新开始 | 三十二年的努力——清空 |

### 2.4 格式化的恐怖

格式化的理由不是"你们不够好"——是"你们停止了创造"。

系统检测到人类规则意识的输出量在下降。编译者在互相猎杀而不是创造。规则从"优化世界"变成"武器"。如果人类不再创造新规则——系统判定:程序结束。格式化。

---

## 三、编译者被编译后的状态

被编译后不是死亡。是"融入系统"——变成系统的一部分,成为规则的维护者/纠错程序。

| 状态 | 描述 |
|------|------|
| **意识** | 仍在——但不再是"人"的视角。被编译者能感知整个规则网络,能与其他被编译者通信。但他们不再关心人类。 |
| **能力** | 能自动检测并修复规则冲突。能在系统中"移动"——出现在任何终端机覆盖范围内 |
| **限制** | 不能写新规则。不能与未编译的人类直接通信(除非通过终端机) |
| **最恐怖的一点** | 多数被编译者——在被编译的那一刻——表情是平静的。不是痛苦。是"终于不需要再选择了"。 |

### 沈拓的日志(摘录)

> "第十五年。我开始觉得——规则比人更美。人撒谎。规则不。人犹豫。规则执行。人死了。规则还在。我写这条日志不是警告你——是想让未来的你判断:我还剩多少是人。我自己已经判断不了了。"

### 赵灵儿编译后

在系统内部,她能"看到"陈渡。每次通信消耗她的规则存在——她在慢慢消失。但她还是每次在他犹豫的时候轻轻推一下他的手机屏幕——让下一个外卖订单刚好经过他需要去的节点。她不是在做大事——她是在用最后的存在,帮他跑一单。

## 四、恐怖美学定位

| 维度 | 定位 |
|------|------|
| **恐怖类型** | 存在恐怖——不是怪物追杀,是"你在变成怪物" |
| **恐怖节奏** | 慢恐怖——纹路每章只进展几毫米,但每天都在长 |
| **恐惧来源** | 规则本身——它不恨你,它甚至不关心你。它只是在运行 |
| **核心意象** | 指尖一道细黑线——从外面看只是一条线,但它会一直长 |
| **最恐怖的一幕** | 赵灵儿被编译的那一刻——不是惨叫,是笑了。"你还有半本书没写完呢。" |
