## 二、世界观设定

### 2.1 核心概念：规则终端机

城市中存在47台终端机——外表像老旧ATM或公司打卡机，分布在写字楼、医院、商场、地铁站。每台覆盖半径3-5公里，形成"规则节点"。所有节点构成一张覆盖全城的规则网络。

终端机不是人造的。它们在三十二年前（1994年）同时出现。

### 2.2 编译者

少数人能"看到"终端机——普通人经过时它只是一堵墙或一台坏电器。这些能看见并操作终端机的人被称为「编译者」。全市编译者约30-50人。

编译者之间天然竞争——每写一条规则，所有编译者都能感知到。规则暴露你的位置。

### 2.3 规则体系

| 类型 | 说明 | 示例 | 风险 |
|------|------|------|------|
| **物理规则** | 改变空间/时间/物理量 | "此楼层重力减半" | 🔴极高 |
| **行为规则** | 触发条件→行为 | "此巷禁止暴力" | 🟡中 |
| **认知规则** | 改变记忆/感知 | "没人记得你昨晚来过" | 🟠高 |
| **元规则** | 修改其他规则 | "此规则优先于所有规则" | 🔴致命 |

层级：局部(单栋楼) < 区域(3-5公里) < 全局(全城)。层级越高代价越大。

### 2.4 代价机制：代码纹路

每写一条规则，身体上出现一行黑色纹路——从指尖向心脏蔓延。

```
指尖 → 手腕 → 前臂 → 肘部 → 肩 → 锁骨 → 心脏
 ↑                                                     ↑
第1条                                          第~30条
```

纹路到心脏时→编译者被"编译"→身体化为代码，意识融入规则网络，成为系统的纠错程序。

### 2.5 规则冲突与裂隙

两条矛盾的规则在同一区域生效→产生"规则裂隙"。裂隙不可预测——扭曲空间、反转时间、抹除存在。

### 2.6 终端机分布（47节点）

四个主节点（可发布全局规则）：城中旧法院、城东电视塔、城南废弃钢厂、城西防空洞。四节点中心——第五个节点："管理员终端"。

## 八、写作规范摘要

- 单章字数:1800-2500字（番茄碎片阅读）
- 每章结尾必须有钩子——信息差/危机/情感悬念
- 叙述:第三人称有限视角(紧跟陈渡)。语气:干的、有细节的、不煽情
- 禁用:过度心理描写、抽象感慨、AI式三段论
- 对话:每轮至少推进一件事(事实/规则/代价三选一)
- 规则描写:必须精确——内容、范围、代价。像产品说明书一样写
